=== Aprimoramentos nas cargas de trabalho do Oracle Database

Nesta versão, as cargas de trabalho do Oracle Database introduzem a seguinte melhoria:

* *Suporte a hosts em cluster*: Agora você pode descobrir e proteger bancos de dados Oracle Real Application Clusters (RAC) que usam Oracle Automatic Storage Management (ASM) e são implantados usando VMDK em sistemas de arquivos VMFS. Para mais informações, consulte https://docs.netapp.com/us-en/data-services-backup-recovery/br-start-discover-oracle.html["Descubra cargas de trabalho do Oracle Database"].
* *Suporte ao catálogo RMAN*: Agora você pode configurar o Backup e Recovery para armazenar metadados de proteção no catálogo de recuperação do RMAN, o que proporciona redundância e retenção de longo prazo caso o arquivo de controle de destino seja perdido. Para mais informações, consulte  https://docs.netapp.com/us-en/data-services-backup-recovery/br-start-configure-oracle-database.html["Configurar um banco de dados Oracle"].


Para obter detalhes sobre como proteger cargas de trabalho do Oracle Database, consulte  https://docs.netapp.com/us-en/data-services-backup-recovery/br-use-oracle-protect-overview.html["Visão geral da proteção de cargas de trabalho do Oracle Database"].
